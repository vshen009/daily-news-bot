# 定制每日财经新闻 - 项目需求文档

## 📋 项目概述

**项目名称**: 定制每日财经新闻
**创建日期**: 2026年1月25日
**项目类型**: 金融资讯日报
**目标用户**: 金融行业从业者、投资者、经济观察者
**更新频率**: 每日

### 项目定位
为金融从业者提供专业的每日新闻汇总，配合AI锐评，快速掌握全球金融动态，提升信息获取效率。

## 📝 内容规范

### 新闻板块划分

#### 一、国内金融 (Domestic Finance)
**关注领域**: 政策法规、监管动态、行业改革、市场开放、金融创新

**示例新闻**:
- 金融监管政策出台/调整
- 央行货币政策动向
- 资本市场改革措施
- 金融科技发展
- 跨境金融合作

#### 二、亚太日本宏观 (Asia Pacific)
**关注领域**: 日本央行政策、亚太股市、外汇市场、债券市场、经济增长

**示例新闻**:
- 日本央行利率决议
- 日元汇率波动
- 亚太主要股市表现
- 各国GDP/通胀数据
- 区域贸易协定

#### 三、美国欧洲 (US & Europe)
**关注领域**: 美欧关系、贸易政策、地缘政治、资本市场、货币政策

**示例新闻**:
- 美欧贸易争端
- 美股资金流向
- 欧洲财政政策
- 地缘政治风险
- 跨境投资趋势

### 每日新闻数量
- **每个板块**: 5条新闻
- **每日总计**: 15条新闻
- **Featured新闻**: 每个板块第1条为重点新闻，占满整行

### 新闻来源规范

**优先级排序**:
1. **权威媒体**: 新华社、人民日报、经济日报
2. **专业财经**: 财新网、第一财经、21世纪经济报道
3. **国际媒体**: Bloomberg、Reuters、WSJ（如有引用）
4. **其他来源**: 地方媒体、行业网站（需注明）

**来源命名规范**:
```
国内: 新华社 XINHUA、财新网 CAIXIN、21世纪经济 21JJ
国际: 新华网 XINHUA、新浪财经 SINA、证券时报 STCN
```

### AI锐评规范

#### 定位
用一句话对新闻进行专业评论，提供金融从业者的洞察视角。

#### 写作原则
1. **专业性**: 使用金融术语和行业黑话
2. **洞察性**: 不仅描述现象，更要揭示背后的逻辑
3. **前瞻性**: 提供对未来趋势的判断
4. **简洁性**: 一句话控制在30-50字

#### 评论维度

**政策新闻**:
- 对市场的影响机制
- 对行业的长期意义
- 投资机会/风险提示

**市场新闻**:
- 驱动因素分析
- 技术面/基本面解读
- 操作建议或策略思路

**事件新闻**:
- 地缘政治含义
- 产业链影响
- 配置建议

#### 示例模板

```
✅ 好的AI锐评:
"费率战本质是存量博弈下的价格竞争，长期看将推动行业向规模效应和投研能力分化，中小公司生存压力加剧。"

❌ 不好的AI锐评:
"这个消息很重要，会影响基金行业。"
```

## 🎬 动画与交互

### 页面加载动画
- **Loading Logo**: 渐变色方块 + "FI" 字母
- **脉冲动画**: 1秒周期，缩放 + 阴影变化
- **进度条**: 滑动的渐变色条
- **淡出时间**: 0.8秒后自动消失

### 核心动画
1. **Header滑入**: 从上方滑入 + 淡入
2. **Section渐显**: 依次向上移动 + 淡入（0.2s/0.4s/0.6s延迟）
3. **章节编号填充**: 渐变色从上到下填充
4. **卡片悬停**:
   - 上浮 4px
   - 光线扫过效果
   - 发光边框
   - 阴影加深
5. **Ticker滚动**: 30秒循环

### 微交互
- **来源链接**: 悬停变色 + 箭头平移
- **状态点**: 呼吸灯闪烁效果
- **数字计数**: 文章数从0滚动到15
- **网格背景**: 持续移动的网格线
- **扫描线**: 从上到下的扫描光效

## 🔄 更新流程

### 每日更新步骤

1. **收集新闻** (上午)
   - 浏览权威媒体金融版块
   - 筛选重要新闻（国内5条 + 亚太5条 + 美欧5条）
   - 记录新闻标题、摘要、来源、链接

2. **撰写AI锐评** (上午)
   - 为每条新闻撰写专业评论
   - 检查评论质量和长度

3. **更新HTML文件** (中午前)
   - 复制前一日文件
   - 重命名为今日日期
   - 替换新闻内容
   - 更新日期变量
   - 检查格式和链接

4. **测试发布** (中午)
   - 本地浏览器预览
   - 检查所有功能和内容
   - 提交到仓库（如需）
